语雀 OpenAPI 开发指南 ​

版本: 2.0.1
基础 URL: https://www.yuque.com
许可证: Apache-2.0

🚀 快速开始 ​

第一步：获取 Token ​

语雀 API 必须使用 Token 进行认证，不支持账号密码认证。

获取开发者 Token ​

  1. 登录 语雀网站
  2. 进入 个人设置 → 开发者设置
  3. 创建新的 Token
  4. 重要：Token 只显示一次，请立即复制并妥善保管

💡 提示：开发者 Token 可能需要付费，请查看语雀官方定价页面。

通过浏览器获取 Token（临时方案） ​

如果暂时无法获取开发者 Token，可以通过浏览器临时获取：

  1. 登录语雀网站
  2. 打开浏览器开发者工具（F12）
  3. 切换到 Network 标签页
  4. 在语雀网站中执行任意操作（如查看文档）
  5. 查看请求头中的 X-Auth-Token 或 Cookie 中的 token 信息
  6. 复制 Token 用于 API 调用

⚠️ 注意：通过浏览器获取的 Token 可能有时效性，建议使用开发者 Token。

第二步：配置 Token ​

方式一：使用环境变量文件（推荐） ​

创建配置文件 yuque_api_config.env：

properties
# 语雀 API 配置文件
# 请将 YOUR_TOKEN_HERE 替换为您的实际 Token

# 语雀 API Token（必需）
YUQUE_API_TOKEN=your_token_here

# 语雀 API 基础地址（可选，默认值）
YUQUE_API_BASE_URL=https://www.yuque.com/api/v2

# 您的团队/用户信息（可选，用于示例）
YUQUE_GROUP_LOGIN=xxxxx
YUQUE_BOOK_SLUG=doc

📖 重要概念 ​

日期和时间格式 ​

语雀 API 使用标准的 ISO 8601 格式，通常为 UTC 时间。

格式示例：

  • 2019-08-24T14:15:22Z - 完整日期时间（UTC）
  • 2019-08-24 - 仅日期

处理建议：

  • 在 JavaScript 中使用 new Date() 解析
  • 在 Python 中使用 datetime.fromisoformat() 解析
  • 注意时区转换（API 返回的是 UTC 时间）

网址路径结构 ​

理解语雀的 URL 结构对于正确使用 API 至关重要。

完整路径示例：

https://www.yuque.com/xxxxx/doc/quick-start
                        |         |    |
                        |         |    +-- doc_slug（文档标识）
                        |         +-- book_slug（知识库标识）
                        +-- group_login（团队/用户登录名）
1

URL 组成部分：

部分说明示例在 API 中的使用
group_login团队或用户的登录名xxxxx路径参数
book_slug知识库的路径标识doc路径参数
doc_slug文档的路径标识quick-start路径参数

常用字段说明 ​

字段类型说明示例获取方式
group_loginstring团队或用户的登录名xxxxxxxxxURL 路径中
book_slugstring知识库的路径标识docURL 路径中
doc_slugstring文档的路径标识quick-startURL 路径中
book_idnumber知识库的数字 ID123456API 响应中
doc_idnumber文档的数字 ID789012API 响应中
user_idnumber用户的数字 ID456789API 响应中

使用建议：

  • slug 系列（group_login、book_slug、doc_slug）：从 URL 中提取，适合不知道 ID 的情况
  • id 系列（book_id、doc_id）：从 API 响应中获取，性能更好，适合批量操作

API 重载（Overload） ​

语雀的部分 API 提供了重载功能，允许使用不同的参数组合访问相同的资源。

示例：获取文档详情 ​

方式一：使用路径参数（推荐用于从 URL 提取）

GET /api/v2/repos/{group_login}/{book_slug}/docs/{doc_slug}
1

适用场景：

  • 已知文档的完整 URL
  • 从网页 URL 中提取参数
  • 不需要先调用其他 API 获取 ID

示例：

bash
# 从 URL: https://www.yuque.com/xxxxxx/doc/quick-start
# 提取参数：group_login=xxxxxxx, book_slug=doc, doc_slug=quick-start

curl -X GET \
  "https://www.yuque.com/api/v2/repos/xxxxxxx/doc/docs/quick-start" \
  -H "X-Auth-Token: YOUR_TOKEN_HERE"

  • 文档中有很多类似的重载接口，请根据实际场景选择

🔐 认证方式 ​

Token 认证（唯一方式） ​

⚠️ 重要：语雀 API 仅支持 Token 认证，不支持账号密码认证（Basic Auth）。

使用方法 ​

方法一：请求头（推荐） ​

在 HTTP 请求头中添加 X-Auth-Token：

bash
curl -X GET \
  "https://www.yuque.com/api/v2/user" \
  -H "X-Auth-Token: YOUR_TOKEN_HERE"
1
2
3

方法二：URL 参数（不推荐） ​

在 URL 查询参数中添加 authToken：

bash
curl -X GET \
  "https://www.yuque.com/api/v2/user?authToken=YOUR_TOKEN_HERE"
1
2

⚠️ 安全提示：URL 参数方式会将 Token 暴露在日志中，不推荐使用。

代码示例 ​

JavaScript/Node.js ​

javascript
// 使用 fetch
fetch('https://www.yuque.com/api/v2/user', {
  headers: {
    'X-Auth-Token': 'YOUR_TOKEN_HERE',
    'Content-Type': 'application/json',
  },
})
  .then((response) => response.json())
  .then((data) => console.log(data))
  .catch((error) => console.error('Error:', error))

// 使用 axios
const axios = require('axios')

axios
  .get('https://www.yuque.com/api/v2/user', {
    headers: {
      'X-Auth-Token': 'YOUR_TOKEN_HERE',
    },
  })
  .then((response) => console.log(response.data))
  .catch((error) => console.error('Error:', error))

Postman 设置 ​

  1. 打开 Postman，创建新请求
  2. 切换到 Headers 标签页
  3. 添加 Header：
    • Key: X-Auth-Token
    • Value: YOUR_TOKEN_HERE（替换为您的实际 Token）
  4. 点击 Send 发送请求

安全最佳实践 ​

  1. 不要硬编码 Token

  2. 使用环境变量

    • 开发环境：使用 .env 文件
    • 生产环境：使用服务器环境变量或密钥管理服务

  3. 不要提交 Token 到代码仓库

    • 将包含 Token 的文件添加到 .gitignore
    • 使用 .env.example 作为模板

  4. 定期轮换 Token

    • 定期更新 Token
    • 发现泄露立即撤销并重新生成

  const match = url.match(/https?:\/\/www\.yuque\.com\/([^\/]+)\/([^\/]+)\/([^\/]+)/)

python
import re
import requests

def parse_yuque_url(url):
    """解析语雀 URL，提取参数"""
    pattern = r'https?://www\.yuque\.com/([^/]+)/([^/]+)/([^/]+)'
    match = re.match(pattern, url)
    if not match:
        return None

    return {
        'group_login': match.group(1),
        'book_slug': match.group(2),
        'doc_slug': match.group(3)
    }

def get_document_from_url(url, token):
    """从语雀 URL 获取文档内容"""
    params = parse_yuque_url(url)
    if not params:
        raise ValueError('Invalid Yuque URL')

    api_url = f"https://www.yuque.com/api/v2/repos/{params['group_login']}/{params['book_slug']}/docs/{params['doc_slug']}"

    headers = {
        'X-Auth-Token': token
    }

    response = requests.get(api_url, headers=headers)
    response.raise_for_status()

    return response.json()['data']

# 使用示例
url = 'https://www.yuque.com/xxxxxxx/doc/quick-start'
token = 'YOUR_TOKEN_HERE'

try:
    doc = get_document_from_url(url, token)
    print(f"文档标题: {doc['title']}")
    print(f"文档内容: {doc['body_html']}")
except Exception as e:
    print(f'Error: {e}')

场景二：批量获取知识库的所有文档 ​

需求：获取某个知识库的所有文档列表，并批量获取文档内容。

实现步骤：

  1. 获取知识库详情，获取 book_id
  2. 获取文档列表
  3. 批量获取文档内容

JavaScript 实现：

javascript
async function getAllDocuments(groupLogin, bookSlug, token) {
  // 1. 获取知识库详情
  const repoResponse = await fetch(`https://www.yuque.com/api/v2/repos/${groupLogin}/${bookSlug}`, {
    headers: { 'X-Auth-Token': token },
  })
  const repo = (await repoResponse.json()).data
  const bookId = repo.id

  // 2. 获取文档列表
  const docsResponse = await fetch(`https://www.yuque.com/api/v2/repos/${bookId}/docs`, {
    headers: { 'X-Auth-Token': token },
  })
  const docs = (await docsResponse.json()).data

  // 3. 批量获取文档内容
  const documents = await Promise.all(
    docs.map(async (doc) => {
      const docResponse = await fetch(
        `https://www.yuque.com/api/v2/repos/${bookId}/docs/${doc.id}`,
        { headers: { 'X-Auth-Token': token } },
      )
      return (await docResponse.json()).data
    }),
  )

  return documents
}

// 使用示例
getAllDocuments('xxxxxxx', 'doc', 'YOUR_TOKEN_HERE').then((docs) => {
  console.log(`共获取 ${docs.length} 个文档`)
  docs.forEach((doc) => {
    console.log(`- ${doc.title}`)
  })
})

实现要点：

JavaScript 实现：

javascript
// 防抖函数
function debounce(func, wait) {
  let timeout
  return function (...args) {
    clearTimeout(timeout)
    timeout = setTimeout(() => func.apply(this, args), wait)
  }
}

// 搜索函数
async function searchYuque(keyword, token, options = {}) {
  const { type = 'doc', offset = 1, limit = 10 } = options

  if (!keyword || keyword.trim().length < 2) {
    return []
  }

  const params = new URLSearchParams({
    q: keyword.trim(),
    type: type,
    offset: offset.toString(),
  })

  try {
    const response = await fetch(`https://www.yuque.com/api/v2/search?${params}`, {
      headers: {
        'X-Auth-Token': token,
        'Content-Type': 'application/json',
      },
    })

    if (!response.ok) {
      throw new Error(`API Error: ${response.status}`)
    }

    const result = await response.json()
    return result.data || []
  } catch (error) {
    console.error('搜索失败:', error)
    return []
  }
}

// 使用示例（Vue.js）
const debouncedSearch = debounce(async (keyword) => {
  const results = await searchYuque(keyword, 'YOUR_TOKEN_HERE')
  // 更新搜索结果
  searchResults.value = results
}, 300)

// 在输入框中监听
watch(searchKeyword, (newValue) => {
  if (newValue && newValue.length >= 2) {
    debouncedSearch(newValue)
  }
})

async function callYuqueAPI(url, token) {

3. 请求频率控制 ​

语雀 API 可能有请求频率限制，建议：

  • 实现请求队列：避免并发请求过多
  • 添加重试机制：对于 429（Too Many Requests）错误，实现指数退避重试
  • 使用缓存：对于不经常变化的数据，使用缓存减少请求

重试机制示例：

javascript
async function callWithRetry(url, token, maxRetries = 3) {
  for (let i = 0; i < maxRetries; i++) {
    try {
      const response = await fetch(url, {
        headers: { 'X-Auth-Token': token },
      })

      if (response.status === 429) {
        // 请求频率过高，等待后重试
        const waitTime = Math.pow(2, i) * 1000 // 指数退避
        await new Promise((resolve) => setTimeout(resolve, waitTime))
        continue
      }

      if (!response.ok) {
        throw new Error(`HTTP ${response.status}`)
      }

      return await response.json()
    } catch (error) {
      if (i === maxRetries - 1) throw error
      await new Promise((resolve) => setTimeout(resolve, 1000 * (i + 1)))
    }
  }
}

5. 分页处理 ​

正确处理分页：

javascript
async function getAllPages(baseUrl, token, limit = 20) {
  const allData = []
  let offset = 1 // 注意：offset 必须 >= 1
  let hasMore = true

  while (hasMore) {
    const url = `${baseUrl}?offset=${offset}&limit=${limit}`
    const response = await fetch(url, {
      headers: { 'X-Auth-Token': token },
    })

    const result = await response.json()
    const data = result.data || []

    allData.push(...data)

    // 判断是否还有更多数据
    hasMore = data.length === limit
    offset += limit
  }

  return allData
}

❓ 常见问题 ​

Q1: 为什么使用账号密码认证返回 401？ ​

A: 语雀 API 不支持账号密码认证（Basic Auth），必须使用 Token 认证。请按照文档获取 Token 并使用 X-Auth-Token 请求头。

Q2: offset 参数为什么必须 >= 1？ ​

A: 语雀 API 的 offset 参数要求必须大于等于 1，不能为 0。这是 API 的设计要求。

Q3: 如何解决跨域（CORS）问题？ ​

A:

  • 开发环境：使用 Vite/VitePress 的代理功能
  • 生产环境：使用后端服务器中转请求

Q4: Token 在哪里获取？ ​

A:

  1. 登录语雀网站
  2. 进入 个人设置 → 开发者设置
  3. 创建 Token（可能需要付费）

Q5: 如何批量获取文档？ ​

A:

  1. 先获取知识库的文档列表
  2. 使用 Promise.all 或循环批量请求文档详情
  3. 注意控制并发数量，避免触发频率限制

Q6: 搜索 API 返回空结果怎么办？ ​

A:

  • 检查搜索关键词是否正确
  • 确认 Token 是否有搜索权限
  • 尝试使用不同的 type 参数（doc/repo/user/all）
  • 检查 offset 参数是否 >= 1

javascript
async function handleApiError(response) {
  if (!response.ok) {
    const error = await response.json().catch(() => ({
      message: `HTTP ${response.status}`,
      status: response.status,
    }))

    switch (response.status) {
      case 401:
        console.error('认证失败：请检查 Token 是否正确')
        // 可以在这里触发重新登录或 Token 刷新
        break
      case 403:
        console.error('权限不足：Token 没有访问该资源的权限')
        break
      case 404:
        console.error('资源不存在：请检查路径参数是否正确')
        break
      case 422:
        console.error('参数错误：', error.message)
        break
      case 429:
        console.error('请求频率过高：请稍后再试')
        break
      default:
        console.error('API 错误：', error.message)
    }

    throw new Error(error.message || `HTTP ${response.status}`)
  }
}
