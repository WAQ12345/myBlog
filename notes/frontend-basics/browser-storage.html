浏览器存储知识体系 ​

存储方式概览 ​

浏览器提供了多种存储方式，每种方式都有其特点和适用场景：

存储方式容量生命周期作用域同步/异步网络请求
Cookie4KB可设置域名+路径同步自动携带
localStorage5-10MB永久域名同步不自动
sessionStorage5-10MB标签页关闭域名+标签页同步不自动
IndexedDB50%+ 磁盘空间永久域名异步不自动
Cache API50%+ 磁盘空间永久域名异步不自动
Web SQL5MB+永久域名异步不自动（已废弃）

Cookie ​

Cookie 概述 ​

Cookie 是服务器发送到浏览器并保存在本地的小型数据片段，浏览器会在每次 HTTP 请求时自动携带 Cookie。

详细内容请参考：网络知识 - Cookie、Session 和 Token

Cookie 特点 ​

  • 自动携带：每次 HTTP 请求自动发送
  • 大小限制：4KB
  • 域名限制：只能被设置 Cookie 的域名访问
  • 网络相关：与 HTTP 协议紧密相关

localStorage ​

localStorage 概述 ​

localStorage 是 HTML5 提供的本地存储 API，数据永久保存在浏览器中，除非手动清除。

localStorage 特点 ​

  • 容量：通常 5-10MB（浏览器差异）
  • 生命周期：永久存储，除非手动清除
  • 作用域：同源策略（协议+域名+端口）
  • 同步操作：所有操作都是同步的，可能阻塞主线程
  • 存储格式：只能存储字符串

封装工具类：

javascript
class LocalStorage {
  static set(key, value) {
    try {
      const serialized = JSON.stringify(value);
      localStorage.setItem(key, serialized);
      return true;
    } catch (error) {
      console.error('localStorage.setItem 失败:', error);
      return false;
    }
  }
  
  static get(key, defaultValue = null) {
    try {
      const item = localStorage.getItem(key);
      if (item === null) {
        return defaultValue;
      }
      return JSON.parse(item);
    } catch (error) {
      console.error('localStorage.getItem 失败:', error);
      return defaultValue;
    }
  }
  
  static remove(key) {
    try {
      localStorage.removeItem(key);
      return true;
    } catch (error) {
      console.error('localStorage.removeItem 失败:', error);
      return false;
    }
  }
  
  static clear() {
    try {
      localStorage.clear();
      return true;
    } catch (error) {
      console.error('localStorage.clear 失败:', error);
      return false;
    }
  }
  
  static has(key) {
    return localStorage.getItem(key) !== null;
  }
  
  static getAll() {
    const result = {};
    for (let i = 0; i < localStorage.length; i++) {
      const key = localStorage.key(i);
      result[key] = this.get(key);
    }
    return result;
  }
}

// 使用
LocalStorage.set('user', { name: 'John', age: 30 });
const user = LocalStorage.get('user');

localStorage 使用场景 ​

  • 用户偏好设置
  • 购物车数据
  • 表单草稿
  • 缓存 API 响应
  • 离线数据

localStorage 注意事项 ​

  1. 容量限制：存储大量数据可能超出限制
  2. 同步操作：大量数据操作可能阻塞主线程
  3. 安全性：不要存储敏感信息（密码、token 等）
  4. 跨标签页：同源的所有标签页共享数据
  5. 隐私模式：某些浏览器的隐私模式可能限制 localStorage

sessionStorage ​

sessionStorage 概述 ​

sessionStorage 与 localStorage 类似，但数据只在当前标签页有效，关闭标签页后数据会被清除。

sessionStorage 特点 ​

  • 容量：通常 5-10MB（浏览器差异）
  • 生命周期：标签页关闭后清除
  • 作用域：同源策略 + 标签页隔离
  • 同步操作：所有操作都是同步的
  • 存储格式：只能存储字符串

IndexedDB ​

IndexedDB 概述 ​

IndexedDB 是一个低级的 API，用于在客户端存储大量结构化数据（包括文件/二进制对象）。

IndexedDB 特点 ​

  • 容量：通常为磁盘空间的 50% 或更多
  • 异步操作：所有操作都是异步的，不会阻塞主线程
  • 支持事务：支持 ACID 事务
  • 索引查询：支持索引和复杂查询
  • 存储类型：可以存储 JavaScript 对象、文件、二进制数据

IndexedDB 基本使用 ​

打开数据库：

javascript
const request = indexedDB.open('myDatabase', 1);

request.onerror = (event) => {
  console.error('数据库打开失败:', event);
};

request.onsuccess = (event) => {
  const db = event.target.result;
  console.log('数据库打开成功:', db);
};

request.onupgradeneeded = (event) => {
  const db = event.target.result;
  
  // 创建对象存储（类似表）
  if (!db.objectStoreNames.contains('users')) {
    const objectStore = db.createObjectStore('users', { keyPath: 'id', autoIncrement: true });
    
    // 创建索引
    objectStore.createIndex('name', 'name', { unique: false });
    objectStore.createIndex('email', 'email', { unique: true });
  }
};

function getUser(db, id) {
  return new Promise((resolve, reject) => {
    const transaction = db.transaction(['users'], 'readonly');
    const objectStore = transaction.objectStore('users');
    const request = objectStore.get(id);
    
    request.onsuccess = () => {
      resolve(request.result);
    };
    
    request.onerror = () => {
      reject(request.error);
    };
  });
}

// 使用索引查询
function getUserByEmail(db, email) {
  return new Promise((resolve, reject) => {
    const transaction = db.transaction(['users'], 'readonly');
    const objectStore = transaction.objectStore('users');
    const index = objectStore.index('email');
    const request = index.get(email);
    
    request.onsuccess = () => {
      resolve(request.result);
    };
    
    request.onerror = () => {
      reject(request.error);
    };
  });
}

Cache API ​

Cache API 概述 ​

Cache API 是 Service Worker 规范的一部分，用于存储 Request/Response 对象对，主要用于缓存网络请求。

Cache API 特点 ​

  • 存储网络响应：可以缓存 HTTP 请求和响应
  • Service Worker：主要用于 Service Worker
  • 异步操作：所有操作都是异步的
  • 支持流：可以缓存流式响应

Cache API 基本使用 ​

打开缓存：

javascript
caches.open('my-cache-v1').then(cache => {
  console.log('缓存打开成功:', cache);
});

Cache API 在 Service Worker 中的应用 ​

javascript
// Service Worker
self.addEventListener('fetch', event => {
  event.respondWith(
    caches.match(event.request)
      .then(response => {
        // 缓存命中
        if (response) {
          return response;
        }
        
        // 缓存未命中，从网络获取
        return fetch(event.request).then(response => {
          // 检查响应是否有效
          if (!response || response.status !== 200) {
            return response;
          }
          
          // 克隆响应（流只能读取一次）
          const responseToCache = response.clone();
          
          // 添加到缓存
          caches.open('my-cache-v1').then(cache => {
            cache.put(event.request, responseToCache);
          });
          
          return response;
        });
      })
  );
});

存储方式选择指南 ​

选择标准 ​

1. 数据大小

  • 小数据（< 5MB）：localStorage、sessionStorage
  • 大数据（> 5MB）：IndexedDB

2. 数据结构

  • 简单键值对：localStorage、sessionStorage
  • 复杂结构化数据：IndexedDB

3. 查询需求

  • 简单查询：localStorage、sessionStorage
  • 复杂查询、索引：IndexedDB

4. 生命周期

  • 永久存储：localStorage、IndexedDB
  • 临时存储：sessionStorage

5. 性能要求

  • 同步操作：localStorage、sessionStorage
  • 异步操作：IndexedDB、Cache API

6. 网络相关

  • 需要自动携带：Cookie
  • 不需要自动携带：其他所有方式

使用场景总结 ​

场景推荐存储方式

总结 ​

浏览器存储是前端开发中的重要知识，选择合适的存储方式可以：

  • 提高应用性能
  • 改善用户体验
  • 支持离线功能
  • 减少服务器压力

关键要点：

  1. Cookie 用于网络请求自动携带的数据
  2. localStorage 用于持久化的简单数据
  3. sessionStorage 用于临时数据
  4. IndexedDB 用于大量结构化数据
  5. Cache API 用于网络请求缓存
  6. 注意数据安全和容量限制
