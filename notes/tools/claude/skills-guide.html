Skills Guide (Cursor & Claude Code) ​

基于 SKILL.md 的技能：教 AI 执行代码审查、commit、查库等专项工作流。

1. Claude Code：存放位置 ​

类型路径作用范围
个人~/.claude/skills/<name>/SKILL.md本机所有项目
项目.claude/skills/<name>/SKILL.md当前项目，可提交 Git

优先级：企业 > 个人 > 项目。子目录 .claude/skills/ 也会被发现（monorepo）。

2. 目录与格式 ​

<name>/
├── SKILL.md       # 必选
├── reference.md   # 可选
├── examples.md    # 可选
└── scripts/       # 可选
1
2
3
4
5

SKILL.md：YAML frontmatter（---）+ Markdown 正文。name 变为 /slash-command；description 供 Claude 判断何时自动加载。

3. Frontmatter（Claude Code） ​

字段说明
name小写/数字/连字符，约 64 字符
description推荐；做什么、何时用
argument-hint参数提示，如 [issue-number]
disable-model-invocationtrue 则仅用户可调用
user-invocablefalse 则不在 / 菜单显示
allowed-tools、context、agent工具限制、子 agent 等

4. 流程：Claude Code 使用项目内 Skills ​

  1. 创建 .claude/skills/<name>/SKILL.md（frontmatter + 正文）。
  2. 在项目根目录执行 claude 或 claude-proxy；技能自动发现，无需写进配置。
  3. 对话中输入 / 查看命令，或 /技能名 调用；Claude 也会按 description 自动加载。
  4. 个人全局：复制到 ~/.claude/skills/。

5. 常见问题 ​

现象处理
403 / Please run /login登录问题，用 claude-proxy 启动后 /login；见 setup
技能未出现在 / 列表确认路径 .claude/skills/<name>/SKILL.md，且从项目根启动；检查 frontmatter
技能列表被截断设置 SLASH_COMMAND_TOOL_CHAR_BUDGET 或精简技能

6. Cursor：存放位置 ​

类型路径
个人~/.cursor/skills/<name>/
项目.cursor/skills/<name>/

不要用 ~/.cursor/skills-cursor/。SKILL.md 格式与 Claude Code 类似（name、description、正文）。

规范与技能内容见 dev-standards。
