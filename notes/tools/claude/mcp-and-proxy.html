MCP & Proxy ​

1. 基本使用 ​

  • 启动：项目根目录执行 claude 或 claude-proxy（需代理时用后者）；首次会询问是否信任工作区。
  • 常用命令：/help、/mcp、/config、/terminal-setup、/status、/exit。

2. MCP 配置位置与作用域 ​

作用域位置说明
user~/.claude.json 顶层 mcpServers本机所有项目可用
local~/.claude.json 按项目路径仅当前项目、仅当前用户
project项目根 .mcp.json可提交 Git，团队共享

优先级：local > project > user。配置写在 ~/.claude.json 或 .mcp.json，不用 ~/.claude/settings.json。

3. MCP 命令（claude mcp） ​

命令说明
claude mcp add [选项] <名称> [URL或-- 命令]添加 MCP
claude mcp list列出已配置 MCP
claude mcp get <名称>查看详情
claude mcp remove <名称>移除
claude mcp add-json [--scope user] <名称> '<JSON>'用 JSON 添加
claude mcp add-from-claude-desktop从 Claude Desktop 导入（macOS/WSL）
claude mcp reset-project-choices重置项目 .mcp.json 启用/禁用选择
claude mcp serve以 stdio MCP 服务器方式启动

add 选项：--transport http|sse|stdio、--scope local|project|user、--env KEY=value、--header "名: 值"。选项写在名称前；stdio 时名称后跟 -- 再跟命令。

对话内：/mcp 查看 MCP 列表与 OAuth 认证。

3.1 安装 MCP（Windows） ​

需代理时先用 claude-proxy.cmd 启动环境，或在 CMD 中先设置代理再执行下列命令。

方式一：命令行添加（filesystem，用户级）

在 CMD 中执行（将 C:\Users\你的用户名 改为你的用户目录，或使用 %USERPROFILE%）：

batch
set HTTP_PROXY=http://127.0.0.1:7890
set HTTPS_PROXY=http://127.0.0.1:7890
claude mcp add filesystem --scope user -- npx -y @modelcontextprotocol/server-filesystem %USERPROFILE%

batch
claude mcp add filesystem --scope user -- npx -y @modelcontextprotocol/server-filesystem %USERPROFILE%

方式二：手动编辑 ~/.claude.json

Windows 下 ~ 对应 %USERPROFILE%（如 C:\Users\你的用户名）。若文件不存在可先运行一次 claude 生成。在顶层增加或合并 mcpServers，例如添加 filesystem：

json
"mcpServers": {
  "filesystem": {
    "type": "stdio",
    "command": "npx",
    "args": ["-y", "@modelcontextprotocol/server-filesystem", "C:\\Users\\你的用户名"]
  }
}

验证：claude mcp list、claude mcp get filesystem；启动 Claude Code 后对话内 /mcp。

启动标志：--mcp-config ./mcp.json、--strict-mcp-config；环境变量 MCP_TIMEOUT、MAX_MCP_OUTPUT_TOKENS、ENABLE_TOOL_SEARCH。

4. 托管代理（企业/网络） ​

  • 代理：HTTPS_PROXY、HTTP_PROXY、NO_PROXY；Basic 认证可写在 URL 中。
  • 自定义 CA：NODE_EXTRA_CA_CERTS=/path/to/ca.pem
  • mTLS：CLAUDE_CODE_CLIENT_CERT、CLAUDE_CODE_CLIENT_KEY、CLAUDE_CODE_CLIENT_KEY_PASSPHRASE
  • 需放行：api.anthropic.com、claude.ai、statsig.anthropic.com、sentry.io
  • LLM Gateway：ANTHROPIC_BASE_URL 等，见官方 Third-party integrations。

5. GitHub MCP Server ​

官方仓库：github/github-mcp-server。支持远程 HTTP（推荐）或本地 Docker/二进制。

前置条件 ​

  • 创建 GitHub Personal Access Token，至少勾选 repo（私有仓库完整权限）；仅用公开库可只勾选 public_repo。
  • Claude Code 2.1.1+ 推荐用远程 HTTP；本地需 Docker 或 发行版二进制。

安全存放 PAT（推荐） ​

避免把 Token 写进配置文件，可用环境变量：

  1. 在项目根或家目录建 .env，内容：GITHUB_PAT=你的Token
  2. 防止误提交：echo -e ".env\n.mcp.json" >> .gitignore（若项目已有 .gitignore 则追加这两行）

远程 HTTP（推荐，无需 Docker） ​

Claude Code 2.1.1+ 在终端执行（不要写进对话里）：

bash
# 用明文 Token（仅本机临时测试）
claude mcp add-json github '{"type":"http","url":"https://api.githubcopilot.com/mcp","headers":{"Authorization":"Bearer YOUR_GITHUB_PAT"}}'

# 用 .env 中的 GITHUB_PAT（项目根下已存在 .env 时）
claude mcp add-json github '{"type":"http","url":"https://api.githubcopilot.com/mcp","headers":{"Authorization":"Bearer '"$(grep GITHUB_PAT .env | cut -d '=' -f2)"'"}}'

作用域（可选）：命令末尾加 --scope user 则本机所有项目可用；不加默认为当前项目（local）。共享给团队用 --scope project（会写入项目 .mcp.json）。

本地 Docker ​

已安装并运行 Docker 时：

bash
# 用明文 Token
claude mcp add github -e GITHUB_PERSONAL_ACCESS_TOKEN=YOUR_GITHUB_PAT -- docker run -i --rm -e GITHUB_PERSONAL_ACCESS_TOKEN ghcr.io/github/github-mcp-server

# 用 .env 中的 GITHUB_PAT
claude mcp add github -e GITHUB_PERSONAL_ACCESS_TOKEN=$(grep GITHUB_PAT .env | cut -d '=' -f2) -- docker run -i --rm -e GITHUB_PERSONAL_ACCESS_TOKEN ghcr.io/github/github-mcp-server

验证与排查 ​

  • 查看列表：claude mcp list
  • 查看 github 配置：claude mcp get github
  • 对话内输入 /mcp 看 MCP 是否就绪；若未出现工具，重启 Claude Code 再试。
  • 认证失败：检查 PAT 是否含 repo、是否过期。
  • 远程不可用：确认 URL 为 https://api.githubcopilot.com/mcp；若需代理，确保该域名被放行或走代理。

详细 MCP 示例与 JSON 结构见官方 MCP、network-config。
