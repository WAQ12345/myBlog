📊 网站访问统计配置 ​

本文档详细介绍如何为 VitePress 博客配置访问统计功能，包括 Google Analytics (GA4) 和百度统计的集成方法。

🎯 功能概述 ​

通过配置访问统计，你可以：

  • ✅ 实时监控：了解网站的实时访问情况
  • ✅ 流量分析：分析访问来源、页面浏览量、用户行为
  • ✅ 数据洞察：了解访客地域、设备类型、浏览器分布
  • ✅ 优化依据：根据数据优化网站内容和用户体验

📋 支持的统计平台 ​

Google Analytics (GA4) ​

  • 🌍 全球最流行的网站分析工具
  • 📊 功能强大，数据详细
  • 🔍 支持实时报告和深度分析
  • ⚠️ 需要科学上网才能查看数据

百度统计 ​

  • 🇨🇳 国内最常用的网站统计工具
  • 🚀 访问速度快，无需科学上网
  • 📈 适合分析国内用户访问情况
  • 🔑 支持搜索引擎关键词分析

🚀 快速开始 ​

第一步：创建环境变量文件 ​

在 VitePress 项目根目录下创建 .env 文件：

bash
cd VitePress

VITE_BAIDU_SITE_ID=xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx

📖 详细配置步骤 ​

配置 Google Analytics (GA4) ​

1. 获取测量 ID ​

  1. 访问 Google Analytics
  2. 使用 Google 账号登录
  3. 创建新的账号（如果还没有）
  4. 创建新的属性（Property）
    • 属性名称：你的网站名称
    • 报告时区：选择你的时区
    • 货币：选择你的货币
  5. 创建数据流（Data Stream）
    • 选择 网站
    • 输入网站 URL
    • 输入流名称
  6. 复制 测量 ID（格式：G-XXXXXXXXXX）

2. 配置环境变量 ​

在 .env 文件中添加：

properties
VITE_GA_MEASUREMENT_ID=G-XXXXXXXXXX
1

3. 验证配置 ​

  1. 构建并部署网站
  2. 访问你的网站
  3. 在 Google Analytics 后台查看 实时 报告
  4. 如果看到访问数据，说明配置成功

配置百度统计 ​

1. 获取站点 ID ​

  1. 访问 百度统计
  2. 使用百度账号登录
  3. 添加网站
    • 网站名称：你的网站名称
    • 网站域名：你的网站域名
    • 网站首页：你的网站首页 URL
  4. 进入 代码管理 -> 代码获取
  5. 选择你的网站
  6. 复制统计代码
  7. 在代码中找到 hm.js? 后面的 ID（32位字符串）
    • 例如：https://hm.baidu.com/hm.js?2c209de3250aed0ad39b4df3cc025494
    • 站点 ID 就是 2c209de3250aed0ad39b4df3cc025494

2. 配置环境变量 ​

在 .env 文件中添加：

properties
VITE_BAIDU_SITE_ID=2c209de3250aed0ad39b4df3cc025494
1

3. 验证配置 ​

方法一：代码安装检查（推荐）

  1. 登录 百度统计后台
  2. 进入 代码管理 -> 代码安装检查
  3. 输入你的网站地址
  4. 点击 自动检测 或 手动检测
  5. 如果显示"代码安装正确"，说明配置成功

方法二：浏览器开发者工具检查

  1. 打开你的网站
  2. 按 F12 打开开发者工具
  3. 切换到 Network（网络） 标签
  4. 刷新页面
  5. 搜索 hm.baidu.com 或 hm.js
  6. 如果看到请求且状态为 200，说明代码加载成功

方法三：查看实时数据

  1. 部署网站后访问你的网站
  2. 在百度统计后台查看 实时访客
  3. 如果看到访问数据，说明配置成功

注意：代码安装正确后，一般需要 20 分钟后 才能在百度统计后台查看到数据。

🔍 代码实现原理 ​

统计代码插入位置 ​

VitePress 会自动将统计代码插入到所有页面的 </head> 标签前，符合统计平台的要求。

Google Analytics 代码 ​

javascript
// 加载 gtag.js
<script async src="https://www.googletagmanager.com/gtag/js?id=G-XXXXXXXXXX"></script>

// 初始化 GA4
<script>
  window.dataLayer = window.dataLayer || [];
  function gtag(){dataLayer.push(arguments);}
  gtag('js', new Date());
  gtag('config', 'G-XXXXXXXXXX', {
    page_path: window.location.pathname + window.location.search
  });
</script>

📊 统计功能说明 ​

Google Analytics (GA4) 功能 ​

  • ✅ 实时报告：查看当前正在访问网站的用户
  • ✅ 页面浏览量：统计每个页面的访问次数
  • ✅ 用户行为：分析用户在网站上的行为路径
  • ✅ 流量来源：了解用户来自哪些渠道（搜索引擎、直接访问、社交媒体等）
  • ✅ 设备统计：分析用户使用的设备类型（桌面、移动、平板）
  • ✅ 地理位置：查看访问者的地理位置分布
  • ✅ 转化跟踪：设置转化目标，跟踪转化率
  • ✅ 自定义事件：跟踪自定义的用户行为

百度统计功能 ​

  • ✅ 实时访客：查看当前正在访问网站的用户
  • ✅ 页面浏览量：统计每个页面的访问次数
  • ✅ 访问来源：分析用户来自哪些渠道
  • ✅ 搜索引擎关键词：查看用户通过哪些关键词找到你的网站
  • ✅ 访客地域：查看访问者的地域分布
  • ✅ 访问设备：分析用户使用的设备类型和浏览器
  • ✅ 访问时段：了解用户访问的高峰时段
  • ✅ 新老访客：区分新访客和回访访客

⚠️ 注意事项 ​

环境变量安全 ​

  1. 不要提交 .env 文件到 Git

    • .env 文件已在 .gitignore 中
    • 统计 ID 虽然不敏感，但建议不要公开

  2. 使用 .env.example 作为模板

    • 项目已包含 .env.example 文件
    • 团队成员可以复制此文件创建自己的 .env

隐私保护 ​

  1. 符合隐私法规

    • 确保统计代码的使用符合 GDPR、CCPA 等隐私法规
    • 考虑添加隐私政策页面

  2. 用户同意

    • 某些地区可能需要在用户同意后才能收集数据
    • 可以考虑添加 Cookie 同意横幅

性能影响 ​

  1. 异步加载

    • 统计代码采用异步加载，不会阻塞页面渲染
    • 对网站性能影响很小

  2. CDN 加速

    • Google Analytics 和百度统计都使用 CDN
    • 加载速度快，全球可用

🔧 常见问题 ​

Q1: 为什么部署后看不到统计数据？ ​

A: 可能的原因：

  1. 环境变量未正确加载：检查 .env 文件是否存在且格式正确
  2. 构建时未加载环境变量：确保在构建前加载了环境变量
  3. 数据延迟：百度统计需要 20 分钟后才能看到数据
  4. 代码未正确插入：使用浏览器开发者工具检查代码是否加载

Q2: 本地开发时统计代码会运行吗？ ​

A: 会的。如果配置了 .env 文件，本地开发时统计代码也会运行。如果不想在本地开发时收集数据，可以：

  • 不创建 .env 文件
  • 或者在 .env 文件中留空对应的 ID

Q3: 可以只使用其中一个统计平台吗？ ​

A: 可以。只需要在 .env 文件中配置你想使用的平台 ID，另一个留空即可。

Q4: 如何禁用统计功能？ ​

A: 有两种方式：

  1. 删除 .env 文件
  2. 在 .env 文件中将对应的 ID 留空

统计代码只有在配置了对应 ID 时才会加载。

Q5: 统计代码会影响网站性能吗？ ​

A: 影响很小。统计代码采用异步加载，不会阻塞页面渲染。如果担心性能，可以：

  • 只使用一个统计平台
  • 使用 Google Analytics 的简化版本
  • 考虑使用自建统计服务

🎉 总结 ​

通过配置访问统计，你可以：

  1. 了解用户行为：知道用户如何访问和使用你的网站
  2. 优化内容：根据数据优化网站内容和结构
  3. 提升体验：了解用户需求，提升用户体验
  4. 分析趋势：跟踪网站访问趋势，制定发展策略

配置完成后，记得定期查看统计数据，根据数据优化你的网站！

💡 提示：建议同时使用 Google Analytics 和百度统计，这样可以更全面地了解网站的访问情况。
