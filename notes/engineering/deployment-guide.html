GitHub Pages 部署指南 ​

本文档介绍如何将 VitePress 博客部署到 GitHub Pages，采用静态文件上传到 Git 仓库的方式。

部署架构 ​

  • 源代码仓库：Gitee（the-road-to-big-front-end）
  • 静态文件仓库：GitHub（myBlog）
  • 部署方式：通过脚本自动构建并推送到 GitHub

部署流程 ​

1. 准备工作 ​

确保已创建 GitHub 仓库用于存放静态文件：

  • 仓库地址：https://github.com/your-username/myBlog.git
  • 分支：main

2. 执行部署脚本 ​

在 VitePress 项目根目录下运行部署脚本：

bash
cd VitePress
bash deploy.sh
1
2

注意：

  • Windows 用户需要安装 Git for Windows 才能使用 Git Bash 运行此脚本
  • 脚本会自动适配 macOS、Linux 和 Windows (Git Bash) 系统

3. 部署脚本说明 ​

deploy.sh 脚本执行以下步骤：

  1. 构建项目：执行 npm run build 生成静态文件
  2. 创建临时目录：用于存放构建产物
  3. 初始化 Git 仓库：在临时目录中初始化新的 Git 仓库
  4. 提交文件：将构建产物提交到本地仓库
  5. 推送到 GitHub：强制推送到 GitHub 的 main 分支
  6. 清理临时文件：删除临时目录

完整脚本代码 ​

在 VitePress 项目根目录创建 deploy.sh 文件，内容如下：

bash
#!/bin/bash

# VitePress 博客部署脚本 - 推送到 GitHub main 分支
echo "🚀 部署 VitePress 博客到 GitHub main 分支..."

# 检查是否在正确的目录
if [ ! -f "package.json" ]; then
    echo "❌ 错误：请在 VitePress 项目根目录运行此脚本"
    exit 1
fi

# 构建项目
echo "📦 构建项目..."
npm run build

# 检查构建输出
if [ ! -d "docs/.vitepress/dist" ]; then
    echo "❌ 构建失败，输出目录不存在"
    exit 1
fi

# 创建临时目录
TEMP_DIR=$(mktemp -d)
echo "📁 创建临时目录: $TEMP_DIR"

# 复制构建文件到临时目录
cp -r docs/.vitepress/dist/* "$TEMP_DIR/"

# 进入临时目录
cd "$TEMP_DIR"

# 初始化 Git 仓库
git init
git add .
git commit -m "Deploy VitePress site to main branch - $(date '+%Y-%m-%d %H:%M:%S')"

# 配置 Git 凭证助手（根据操作系统自动选择）
if [[ "$OSTYPE" == "darwin"* ]]; then
    # macOS
    git config credential.helper osxkeychain
elif [[ "$OSTYPE" == "msys" || "$OSTYPE" == "cygwin" ]]; then
    # Windows (Git Bash)
    git config credential.helper manager-core
else
    # Linux
    git config credential.helper store
fi

# 添加 GitHub 远程仓库
git remote add origin https://github.com/your-username/myBlog.git

# 推送到 main 分支
echo "🌐 推送到 GitHub main 分支..."
git push -f origin HEAD:main

# 清理临时目录
cd "$OLDPWD"
rm -rf "$TEMP_DIR"

echo "✅ 部署完成！"
echo "🌐 访问地址：https://your-username.github.io/myBlog/"

使用说明：

  1. 将脚本中的 your-username 替换为你的 GitHub 用户名
  2. 将脚本中的 your-repo 替换为你的 Gitee 仓库名
  3. 给脚本添加执行权限：chmod +x deploy.sh（macOS/Linux）
  4. 执行脚本：bash deploy.sh

4. 配置说明 ​

base 路径配置 ​

在 docs/.vitepress/config.js 中，需要设置正确的 base 路径：

javascript
base: process.env.NODE_ENV === 'production' ? '/myBlog/' : '/',
1

如果 GitHub Pages 仓库名不是 myBlog，需要相应修改此配置。

资源路径 ​

所有静态资源（图片、PDF 等）应放在 docs/public/ 目录下，构建时会自动复制到输出目录。

5. 访问地址 ​

部署完成后，访问地址为：

  • GitHub Pages：https://your-username.github.io/myBlog/

6. 更新内容 ​

每次更新内容后，只需重新运行部署脚本即可：

bash
bash deploy.sh
1

脚本会自动：

  • 构建最新内容
  • 推送到 GitHub
  • 更新 GitHub Pages 站点

注意事项 ​

  1. Git 认证：确保本地已配置 GitHub 访问凭证（SSH 或 HTTPS token）
    • macOS：使用 osxkeychain 凭证助手
    • Windows (Git Bash)：使用 manager-core 凭证助手（Windows 凭证管理器）
    • Linux：使用 store 凭证助手
  2. 强制推送：脚本使用 git push -f，会覆盖远程仓库的 main 分支
  3. 构建产物：docs/.vitepress/dist/ 目录包含所有静态文件
  4. 缓存问题：如果部署后页面未更新，可能是浏览器缓存，尝试清除缓存或使用无痕模式
  5. Windows 用户：需要安装 Git for Windows 才能使用 Git Bash 运行脚本

故障排查 ​

404 错误 ​

如果访问出现 404，检查：

  1. GitHub 仓库 Settings → Pages 中是否已启用 GitHub Pages
  2. Source 是否设置为 main 分支
  3. base 路径配置是否正确

资源加载失败 ​

如果图片、样式等资源加载失败：

  1. 检查资源文件是否在 docs/public/ 目录
  2. 检查资源路径是否正确（注意 base 前缀）
  3. 查看浏览器控制台的错误信息

部署脚本失败 ​

如果脚本执行失败：

  1. 检查是否在正确的目录（VitePress 项目根目录）
  2. 检查 package.json 是否存在
  3. 检查构建是否成功（docs/.vitepress/dist/ 是否存在）
  4. 检查 Git 远程仓库配置是否正确
